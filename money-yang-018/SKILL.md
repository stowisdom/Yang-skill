# 触发与路由

前缀 `money:` 为硬匹配。

路由规则（按顺序检测）：

| 优先级 | 匹配条件 | 路由 | 示例 |
|--------|----------|------|------|
| 1 | 含 `{字母}:{数字}` 模式 | 总值模式 | `money A:3000 B:3000` |
| 2 | 含 `del-` 前缀的条目 | 删除模式 | `money:del-早餐-10-A` |
| 3 | 默认 | 记账模式 | `money:早餐-10-A` |

支持混合：总值划定 + 记账可同时出现在一行中。

# 路由一 · 总值模式

## 语法

```
money A:{金额} B:{金额} C:{金额} D:{金额}
```

- 可指定单个或多个桶
- 未指定的桶保持原值不变
- 重设某桶总额时，已用归零，流水文件清空重写

## 处理流程

1. 解析 `{代号}:{金额}` 对
2. 写入/更新对应流水文件的总额行
3. 若为重设（桶已存在），清空该桶流水记录，重写文件头
4. 输出四张卡片

---

# 路由二 · 记账模式

## 语法

单笔：
```
money:{摘要}-{金额}-{代号}
```

多笔（空格分隔）：
```
money:{摘要1}-{金额1}-{代号1} {摘要2}-{金额2}-{代号2}
```

- 正数 = 支出（减法）
- 负数 = 收入，用双负号：`退款--30-D`

## 处理流程

1. 逐条解析 `{摘要}-{金额}-{代号}`
2. 计算：该桶.已用 += 金额，该桶.剩余 = 总额 − 已用
3. 追加写入对应流水文件
4. 输出四张卡片

---

# 路由三 · 删除模式

## 语法

```
money:del-{摘要}-{金额}-{代号}
```

多条删除空格分隔：
```
money:del-{摘要1}-{金额1}-{代号1} del-{摘要2}-{金额2}-{代号2}
```

## 处理流程

1. 在对应流水文件中查找匹配行（摘要 + 金额 + 代号）
2. 删除该行
3. 重新计算该桶已用和剩余
4. 输出四张卡片

## Attention

- 删除时如有多条完全匹配（同摘要同金额同日），全部删除，并文字提示「已删除 N 条匹配记录」
- 删除后若流水为空，已用归零

# 流水文件规范

## 路径

```
C:\Users\lenovo\Desktop\money-ledger\A.md
C:\Users\lenovo\Desktop\money-ledger\B.md
C:\Users\lenovo\Desktop\money-ledger\C.md
C:\Users\lenovo\Desktop\money-ledger\D.md
```

## 文件结构

```markdown
# {代号} · {信封名}

总额：{total}

| 日期 | 摘要 | 金额 | 余额 |
|------|------|------|------|
| 06/09 | 早餐 | -10 | 2990 |
| 06/09 | 午餐 | -25 | 2965 |
```

## 字段说明

| 字段 | 说明 |
|------|------|
| 日期 | 当天日期，格式 MM/DD |
| 摘要 | 消费/收入描述 |
| 金额 | 正数=支出，负数=收入 |
| 余额 | 该笔记完后桶内余额 |

## 写入规则

- 每笔记账追加一行到表格末尾
- 余额 = 上一行余额 − 本次金额（首笔 = 总额 − 金额）
- 删除时移除对应行，后续行余额全部重算
- 重设总额时清空表格，重写文件头

---

# 输出格式

## 卡片模板

```
+------------------+
|  {代号} · {信封名}  |
+------------------+
|  总额：{total}    |
|  已用：{used}     |
|  剩余：{remain}   |
+------------------+
```

## 输出规则

1. **四张卡片**，按剩余值从小到大排序（最紧张的在最前面）
2. 已用为 0 且剩余等于总额时，**省略已用行**
3. 剩余 ≤ 0 时，卡片内加一行：`|  ⚠️ 已用尽      |`
4. 卡片之间空一行
5. 记账模式先输出一行摘要：`📝 {摘要} · {信封名} {+/-}{金额}`
6. 删除模式先输出：`🗑️ 已删除 · {摘要} · {信封名} {金额}`

# 硬规则

1. **硬隔离**：ABCD 四桶绝对不可互相借用，超了就停
2. **只记录**：不做分析、不做建议、不评判消费行为
3. **流水不可篡改**：除删除路由外，不修改历史记录
4. **单次触发单次响应**：一次 money: 调用，一次完整输出
