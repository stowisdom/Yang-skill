## 核心概念

### 信封代号

| 代号 | 信封名 | 典型占比 |
|------|--------|----------|
| A | 生活 | 30% |
| B | 学习 | 30% |
| C | 储值 | 20% |
| D | 消费 | 20% |

占比仅为建议，用户自行决定各桶总额。

### 硬规则

1. **硬隔离**：四桶之间绝对不可互相借用，某桶用完即停
2. **只记录**：不做分析、不做建议、不评判消费行为
3. **流水不可篡改**：除删除路由外，不修改历史记录

## 输入格式

触发前缀 `money:` 后接内容，共三种路由：

### 路由一 · 总值模式 — 设定桶的总额

```
money A:3000
money A:3000 B:3000 C:2000 D:2000
```

- 可指定单个或多个桶，空格分隔
- 未指定的桶保持原值不变
- 重设某桶总额时，已用归零，该桶流水清空

### 路由二 · 记账模式 — 记录收支

单笔：

```
money:早餐-10-A
```

多笔（空格分隔）：

```
money:早餐-10-A 午餐-25-A 课程-299-B
```

收入（退款）用双负号：

```
money:退款--30-D
```

- 正数金额 = 支出（从桶中扣除）
- 负数金额 = 收入（加回桶中）

### 路由三 · 删除模式 — 删除已记条目

```
money:del-早餐-10-A
```

多条删除空格分隔：

```
money:del-早餐-10-A del-午餐-25-A
```

- 匹配条件：摘要 + 金额 + 代号
- 多条完全匹配时全部删除，提示删除条数

### 混合操作

总值划定 + 记账可同时出现：

```
money A:5000 早餐-10-A
```

### 代号大小写

不敏感，`a` = `A`，`b` = `B`。

## 输出格式

### ASCII 卡片

每个信封输出为一张独立卡片，四张卡片按剩余值从小到大排序（最紧张的在最前面）。

```
+------------------+
|  A · 生活        |
+------------------+
|  总额：3000      |
|  已用：2910      |
|  剩余：90        |
+------------------+
```

### 输出规则

| 规则 | 说明 |
|------|------|
| 已用 = 0 | 省略已用行，只显示总额和剩余 |
| 剩余 ≤ 0 | 卡片内加 `\|  ⚠️ 已用尽` |
| 记账模式 | 卡片前输出一行：`📝 {摘要} · {信封名} +/-{金额}` |
| 删除模式 | 卡片前输出一行：`🗑️ 已删除 · {摘要} · {信封名} {金额}` |
| 卡片间距 | 卡片之间空一行 |

## 流水账本规范

### 文件结构

每个代号一个独立 `.md` 文件，统一存放在 `C:\Users\lenovo\Desktop\money-ledger\`：

```markdown
# A · 生活

总额：3000

| 日期 | 摘要 | 金额 | 余额 |
|------|------|------|------|
| 06/09 | 早餐 | -10 | 2990 |
| 06/09 | 午餐 | -25 | 2965 |
| 06/09 | 房租 | -2900 | 65 |
```

### 字段

| 字段 | 说明 |
|------|------|
| 日期 | 当天日期，格式 `MM/DD` |
| 摘要 | 消费/收入的简短描述 |
| 金额 | 正数 = 支出，负数 = 收入 |
| 余额 | 该笔记完后桶内剩余金额 |

### 写入规则

- 每笔记账追加一行到表格末尾
- 余额 = 上一行余额 − 本次金额（首笔 = 总额 − 金额）
- 删除时移除对应行，后续行余额全部重算
- 重设总额时清空表格，重写文件头
