# ddltodo: — 写入 DDL Todo List

触发前缀 `ddltodo:` 为硬匹配，必须以此前缀开头才触发本 subskill。

---

## 功能

将用户输入的任务文本追加写入 `ddl-yang-002/todolist.txt` 文件。

---

## 工作流

### Step 1 · 解析输入

用户输入 `ddltodo:` 后跟任务文本。

提取任务文本内容。

### Step 2 · 格式校验

检查任务格式是否符合 `任务名--时间` 的格式：

```
正确格式：
任务名--M/D
任务名--M-D

示例：
机械原理作业--4/8
计算机绘图&autocad--4/9
```

如果格式不正确，提示用户正确格式。

### Step 3 · 追加写入

将任务文本追加写入 `todolist.txt` 文件。

写入格式：
```
{任务名}--{时间}
```

### Step 4 · 排序

写入后，读取 todolist.txt 全部内容，按截止时间从近到远排序，重写文件。

排序逻辑：
1. 解析每行：`任务名--M/D`
2. 计算每行的截止日期（当年的 M/D）
3. 按截止日期从近到远排序
4. 保留注释行（`#` 开头）在文件顶部
5. 跳过空行
6. 重写 todolist.txt

### Step 5 · 确认输出

输出确认信息：

```
✅ 已写入 todolist.txt：
{任务名}--{时间}

当前待办任务数：{N} 个
最近截止：{#1 任务名}（{M/D}）
```

## 注意事项

1. 追加写入，不覆盖已有内容
2. 每行一个任务
3. 跳过空行和注释行（以 # 开头）
4. 自动去除首尾空格
