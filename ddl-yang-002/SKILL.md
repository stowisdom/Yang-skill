# DDL Skill - 任务截止时间管理

## 功能特性

- ⏰ **统一时间单位**：全部以「天」计算，无分类
- 📐 **开根号节点**：自动计算两个时间标尺节点（√总天数、√剩余天数），量化行动节奏
- 🔢 **编号排序**：按剩余天数从近到远排序，编号 01, 02, 03...
- 🏮 **东方极简美学**：「墨」风格深色主题，单色墨金，留白、内敛、意蕴丰富
- 🔍 **时间焦点**：视觉重心转向时间，节点超大号、截止大号、事项常规，一目了然

## 触发场景

用户发送"DDL"（三个字母，不区分大小写）时触发本技能。
以下表达也应触发：提到截止日期、deadline、任务到期、任务进度等。
用户粘贴任务列表文本（PDF/MD/TXT内容均可）后触发处理。

### 触发模式

#### 模式A：`ddl`（无内容）
当用户仅输入 `ddl` 或 `DDL`（不带冒号和后续内容）时：
1. 自动读取同目录下的 `todolist.txt` 文件
2. 将文件中的任务列表作为本次处理的全部输入
3. 按标准流程生成可视化报告

#### 模式B：`ddl:内容`（带内容）
当用户输入 `ddl:` 后跟任务文本时：
1. 自动读取同目录下的 `todolist.txt` 文件
2. 将文件中的任务与用户输入的任务合并（去重）
3. 按标准流程生成可视化报告

### 前置动作：读取 todolist.txt

每次触发时（无论模式A还是模式B），先读取同目录下的 `todolist.txt` 文件。

读取逻辑：
1. 检查 `todolist.txt` 文件是否存在（路径：`C:\Users\lenovo\Desktop\todolist.txt`）
2. 若文件存在，读取内容，解析每行任务（格式：`任务名--时间`）
3. 跳过空行和注释行（以 `#` 开头）
4. 若为模式B，将解析出的任务列表与用户当前输入的任务合并
5. 去重（以任务名为准，保留最新时间）
6. 若文件不存在且为模式A，提示用户先用 `ddltodo:` 添加任务

合并后的任务列表作为本次处理的输入。

## 工作流

<work>
	<step1>
	自动获取当前时间：
	通过 date 命令获取当前时间
	令 <now> = 系统返回的当前时间
	若用户在消息中明确指定了其他时间，则以用户指定的时间覆盖 <now>
	</step1>
	
	<step2>
	接收并解析用户输入的任务文本。

	格式要求：每行一个任务，格式为 <任务名>--<截止时间>
	截止时间格式支持：M/D（如 4/8）、M-D（如 4-8）

	从用户输入中提取所有任务条目，过滤空行，得到<条目列表>。

	格式校验：
	- 检查是否有重复任务名，若有则提示用户
	- 统计有效任务数
	</step2>
	
	<step3>
	统一以「天」为单位计算时差：
	做减法：用<ddl>减去<now>得到<时差>
	令 <时差天数> = <时差> 的天数（保留一位小数）
	令 <时差小时> = <时差> 的小时数（取整）
	</step3>

	<step3.5>
	开根号节点计算（时间标尺）
	对每个任务，计算两个关键时间节点：

	■ 当 时差天数 ≥ 2天 时：
	令 <总天数> = 时差天数（四舍五入取整）

	第一节点：
		<N1天数> = floor(√<总天数>)  （取小不取大，即向下取整）
		<第一节点日期> = <now> + <N1天数>天

	第二节点：
		<剩余天数> = <总天数> - <N1天数>
		<N2天数> = floor(√<剩余天数>)  （取小不取大）
		<第二节点日期> = <第一节点日期> + <N2天数>天

	示例：
		today=4/15, ddl=4/24, 总天数=9
		第一节点：√9 = 3 → 4/18
		第二节点：√(9-3)=√6 ≈ 2.449 → floor=2 → 4/18+2 = 4/20

	节点日期格式：显示「M/D」（如 4/18）

	边界处理：
	- 时差天数 < 2天 → 不计算节点
	- N1 = 0 → 不显示节点
	- N2 = 0 → 仅显示第一节点
	</step3.5>
	
	<step4>
	按剩余天数排序（近→远），并分配编号：
	- 编号格式：01, 02, 03...
	- 排序依据：<时差天数> 从小到大
	- 相同天数的任务按任务名字母序排列
	</step4>
	
	<step5>
	整理为一份完整单文件HTML报告
	应用「墨」风格审美设置（详见下文）
	以进度条的形式显示各项任务时间（单色墨金）

	视觉焦点转向时间：
	每张任务卡片内，设「时间焦点区」，通过字体大小建立视觉层级：
	- 编号：1.5rem / font-weight:600 / --ink-dim
	- 任务名称：1.2rem / font-weight:600 / --white
	- 第一、第二节点日期：超大号（2.8rem / font-weight:700 / --ink）
	- 截止日期：大号（2rem / font-weight:600 / --white）
	- 剩余天数：0.88rem / --text
	节点区（左侧）与截止日期区（右侧）通过 flex 布局左右对齐，形成视觉对比呼吸感
	无节点的任务（时差天数 < 2天）仅显示大号截止日期
	</step5>

</work>

## 东方极简「墨」风格审美设置

### 风格定位
东方极简「墨」——深色、留白、内敛、意蕴丰富，单色墨金

### 颜色系统（CSS变量）
```css
--bg: #0a0a0a           /* 近黑背景 */
--surface: #111         /* 表面色 */
--border: #1e1e1e       /* 边线 */
--border-light: #2a2a2a /* 浅边线 */
--text: #a09888         /* 正文暖灰 */
--text-muted: #605848   /* 次要文字 */
--text-dim: #383028     /* 最淡文字 */
--white: #e8e0d4        /* 亮色文字 */
--ink: #c4a87c          /* 墨金强调（主色） */
--ink-dim: #7a6848      /* 淡墨金 */
```

### 字体系统（Google Fonts）
- 中文正文/标题：Noto Serif SC（宋体衬线）
- 中文标签/小字：Noto Sans SC（无衬线）
- 英文点缀：Cormorant Garamond（斜体衬线）

### 字号层级
- 装饰大字：7rem / font-weight:900 / 颜色接近背景色(--border-light)，半隐半现
- 编号：1.5rem / font-weight:600 / --ink-dim
- 卡片标题：1.2rem / font-weight:600 / --white
- 正文：0.88rem / font-weight:400 / --text / line-height:2.2 / text-indent:2em
- 标签小字：0.55-0.7rem / font-weight:300 / --text-dim / letter-spacing:0.3em+

### 时间焦点区（卡片内组件）
卡片内视觉焦点转向时间，通过字号差异建立时间层级：
- **节点日期**（第一、第二节点）：2.8rem / font-weight:700 / --ink（最醒目）
- **截止日期**：2rem / font-weight:600 / --white（次醒目）
- **任务名称**：1.2rem / font-weight:600 / --white（常规层级）
- **编号**：1.5rem / font-weight:600 / --ink-dim（辅助标识）

布局：节点区左对齐，截止日期右对齐，形成左右呼吸对比
节点下方标注小字标签（「第一节点」「第二节点」/ 0.55rem / --text-dim / letter-spacing:0.3em）
节点下方附开根号公式（Cormorant Garamond / italic / 0.75rem / --text-muted），如「√9 = 3日」
无节点任务（时差天数 < 2天）仅显示 2.2rem 截止日期，左对齐

### 布局规则
- 最大宽度800px，居中
- 充足的padding（3-4rem），用留白制造呼吸感
- 用1px solid var(--border) 做段落间隔离线，制造层级
- 竖排文字（writing-mode:vertical-rl）做侧边装饰

2. **标题区**——居中：超大装饰汉字 + 主标题 + 英文副标题 + 小圆点

## Pitfalls

### Windows strftime 格式不兼容
- `%-m` / `%-d`（Linux 零抑制格式）在 Windows 上抛出 `ValueError: Invalid format string`
- **解决方案**：用 `f"{dt.month}/{dt.day}"` 手动拼接，不依赖 strftime
- `%#m` 是 Windows 等价写法，但跨平台代码建议用手动拼接更可靠
