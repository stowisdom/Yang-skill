# information-showing-yang-007

触发前缀 `show:` 为硬匹配，必须以此前缀开头才触发本 skill。

## Step 1
用户输入 `show:` 后跟待处理文本（.txt或.md格式）

## Step 2 可视化

### 绝对禁止：对原文进行增删改，只是按以下规则对文本内容进行可视化处理，辅助阅读

**唯一允许的操作：** 添加 CSS 样式、添加 HTML 标签包裹、调整间距/颜色/字体/字号。

**绝对不允许的操作：** 删除任何元素、合并任何段落、重编号、改变层级。

判断标准：每一段原文必须对应 HTML 中的一个独立单元；输出的总字数（纯文字，不含标签）必须 ≥ 原文的总字数。

### Visual Design System · Napkin Dark

**主题与氛围：** 深色有机主题，背景色取墨绿底黑（#0c0c0a），卡片面与背景仅微差（#131310），制造悬浮感而非强对比。整体气质：克制、呼吸感、线条感、高级感。禁止使用纯黑 #000000 或纯白 #ffffff。

**字体系统：** 正文使用 Noto Serif SC（weight 300），衬线体，轻盈不厚重。标题与关键词使用 Playfair Display（italic），两者同为衬线但气质有别。

**色彩系统：**

| 用途 | 色值 | 说明 |
|------|------|------|
| 大背景 | #0c0c0a | 墨绿底黑 |
| 卡片面 | #131310 | 与背景微差 |
| 正文色 | #c8c3b6 | 亚麻白 |
| 「」关键词 | #b08a4a | 铜金 |
| <>关键词 | #7ab8a0 | 青瓷绿 |
| 引言线/脚注线 | #8b3a3a | 酒红 |
| 折痕线 | #4a4232 | 暖米灰，两端渐变透明 |
| 辅助弱化文字 | #5a5a50 | 引言块文字色 |

**文字处理规则：** 模型输出的每句话都必须用 `<>`、`()`、`「」`、`{}` 进行对重要语义单元的框定强调，减少文字处理的视觉负荷。

**餐巾纸（Napkin）布局结构：**
每张餐巾纸是一个独立的 `<article class="napkin">` 卡片，代表一个完整的文意单元。卡片之间间距 3px。

每张卡片内部结构依次为：
1. 顶部折痕线 — 横贯全宽、两端渐变透明的暖米灰线
2. 右上角编号 — No. 01 格式
3. 斜体标题 — Playfair Display，1.5rem
4. 正文段落 — 含标注系统
5. 引言块（可选）— 酒红左线
6. 底部脚注 — 酒红顶线

### 转换规则（逐段对应，不允许判断性合并）

原文的每一个标题、段落、列表、代码块、表格都是独立单元，必须一一对应生成 HTML 元素：

- 原文的 `#` → HTML 的 `h2`，`##` → `h3`，`###` → `h4`，`####` → `h5`，按原文层级一一对应，不能因为"看起来层级太深"而降级
- 每一段原文 → 对应一个 HTML 单元，不能因为"内容很少"而合并到父级
- 不允许跨段合并、不允许重排段落顺序

### 禁止项
- 禁止使用无衬线字体作为正文
- 禁止使用纯色直线作为分割线（须两端渐变）
- 禁止将人名、工具名放入「」，禁止将理论名放入 <>
- 禁止在同一段落中出现超过 3 个标注词
- 禁止使用灰色系作为任何可读文字的颜色

### PITFALL — "不增删改"的严格执行标准（踩坑教训）

"不增删改原文内容"不仅是文字内容，还包括以下结构元素：

1. **编号体系必须原样保留**：原文的 `B0-1-1-1`、`B2-1-2-3` 等多级编号必须在 HTML 中原样出现，不得简化为 `B0-2-1`、`B2-1-1` 等"看起来更整齐"的编号。编号是层级结构的标识符，改动编号 = 改动结构。

2. **层级标签必须原样保留**：原文中的 `<孙层：xxx>`、`{概念名}`、`「术语」` 等标签必须原封不动出现在 HTML 中。不得删除"孙层"标签、不得将"孙层"改写为"子层"或省略。

3. **不得为排版美观而合并层级**：原文中孙层是独立的标题+列表项，HTML 中也必须是独立的 `h4` + 列表。不得将 3 个孙层压缩进 1 个 `h4` 下合并呈现。每个孙层是独立的知识单元，合并 = 丢失层级感。

4. **不得为"精简"而删除任何层级的内容**：即使某个孙层只有 2-3 句话，也必须完整保留。删除任何层级的任何内容都违反"不增删改"约束。

5. **验证方法**：生成 HTML 后，输出前必须执行以下验证：
   - 用 `grep -o 'B[0-9]*-[0-9]*-[0-9]*-[0-9]*' file.html | sort -u` 检查孙层编号是否完整
   - 用 `grep -c '孙层' file.html` 检查孙层标签数量
   - 比对原文的标题层级树与 HTML 的标题层级树是否一致（每个 `####` 在 HTML 中是否仍有对应的 `h5`）
   - 统计原文纯文字字数与 HTML 纯文字字数，后者必须 ≥ 前者
   - 任何验证失败，必须修复后重新验证，不得输出未通过验证的文件
